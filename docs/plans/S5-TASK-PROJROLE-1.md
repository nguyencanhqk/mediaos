## §Ma trận D-24 (nguồn duy nhất — ADR mirror bảng này)

Áp dụng KHI actor với tới tài nguyên **chỉ nhờ membership** (data_scope của pair < Company). Scope Company/System bypass tầng role (SPEC-06 §18.6.8 'permission hệ thống là lớp kiểm soát cao nhất'). Nhánh **assignee** (task được giao cho chính mình) không bị cap — đó là đường Own truyền thống.

| Hành động trong dự án | Viewer | Member | Manager | Owner |
| --- | --- | --- | --- | --- |
| Xem project/task/board/members/comments/files/checklist | ✓ | ✓ | ✓ | ✓ |
| Watch (tự theo dõi) | ✓ | ✓ | ✓ | ✓ |
| Viết comment · tick checklist · upload file task | ✗ | ✓ | ✓ | ✓ |
| Sửa task ĐƯỢC GIAO cho mình (update/status/state) | (nhánh assignee — role không cap) | ✓ | ✓ | ✓ |
| Sửa/move/assign/priority/deadline task NGƯỜI KHÁC | ✗ | ✗ | ✓ | ✓ |
| Tạo task trong dự án | ✗ | ✗ | ✓ | ✓ |
| Quản lý cột pipeline (project_state CUD) | ✗ | ✗ | ✓ | ✓ |
| Sửa thông tin dự án (field thường) | ✗ | ✗ | ✓ | ✓ |
| Quản lý thành viên · đổi chủ · close/archive/delete dự án | ✗ | ✗ | ✗ | ✓ |
| `project_role` NULL (member legacy user_id-only) | — | = Member (read/collab; KHÔNG write-rộng/govern) | — | — |

Vẫn cần pair hệ thống tương ứng ở lớp 1 (PermissionGuard) — role per-project KHÔNG thay thế permission (SPEC-06 §14.6, API-06 §6.3). Mode ép ở predicate: `read` / `collab` / `write`, **thread theo public method** (một helper phục vụ cả đọc lẫn ghi thì nhận mode từ caller — xem lane be-core (3)).
